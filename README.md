3. Kiro :https://github.com/jasonkneen/kiro
Kiro (Amazon): An IDE-focused, spec-driven tool that emphasizes building AWS-native applications, often used for more formal, structured workflows.
System Prompts for Kiro IDE by Amazon - A comprehensive guide to systematic feature development using the three-phase spec process: Requirements → Design → Tasks.

4. Agent OS is a system for injecting your codebase standards and writing better specs for spec-driven development.

OpenSpec (Fishian AI): A lightweight alternative that focuses on simplicity, using three main steps—proposal, review, and implementation—to prevent AI from losing requirements. It is designed for quick, iterative development, often preferred for brownfield scenarios. OpenSpec is generally faster and simpler to set up.

Augment Code: An enterprise-focused tool that maintains a persistent architectural understanding across very large codebases, making it superior for scaling specs across hundreds of thousands of files.
